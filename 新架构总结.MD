启动服务
Bash

# 1. 启动基础设施（如果 minio 已在运行，可以只启动 postgres + redis）
cd /root/workspaces/rjcut
docker-compose up -d postgres redis minio

# 2. 初始化数据库
python scripts/init_db.py

# 3. 创建测试商户
python scripts/create_merchant.py --name "TestCompany" --quota 1000

# 4. 启动 API
uvicorn api_service:app --host 0.0.0.0 --port 8000 --reload

# 5. 启动 Worker（另一个终端）
python worker.py

# 6. 启动 Watchdog（另一个终端）
python task_watchdog.py
或者 Docker 一键启动
Bash

docker-compose up -d --build

# 创建商户
docker exec rjcut_api python scripts/create_merchant.py --name "TestCompany" --quota 1000
API 调用示例
Bash

# 保存创建商户时得到的 API Key
API_KEY="rjk_xxxx..."
BASE_URL="http://localhost:8000"
ADMIN_KEY="change_me_in_production_2024"

# ── 1. 查看商户信息 ──
curl -s "$BASE_URL/v1/merchant/info" \
  -H "Authorization: Bearer $API_KEY" | python -m json.tool

# ── 2. 预签名上传视频 ──
curl -s -X POST "$BASE_URL/v1/uploads/presign" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "filename": "test.mp4",
    "content_type": "video/mp4",
    "purpose": "input"
  }' | python -m json.tool

# 拿到 upload_url 后用 curl PUT 上传：
# curl -X PUT "<upload_url>" --upload-file test.mp4

# ── 3. 确认上传 ──
curl -s -X POST "$BASE_URL/v1/uploads/confirm" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"upload_id": "<upload_id>"}' | python -m json.tool

# ── 4. 提交任务（使用 OSS key 或 URL） ──
curl -s -X POST "$BASE_URL/v1/tasks/agent-compose" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "input": {
      "video_url": "<oss_key_or_url>",
      "script_url": "<oss_key_or_url>"
    },
    "pipeline": {
      "remove_keyword": "转场",
      "use_transitions": false
    },
    "asr": {
      "model": "large-v3",
      "device": "cuda"
    },
    "subtitle": {
      "effect": "ad",
      "font_size": 88
    },
    "callback": {
      "url": "https://your-server.com/callback",
      "secret": "your_secret"
    },
    "timeout_seconds": 1800
  }' | python -m json.tool

# ── 5. 查询任务状态 ──
curl -s "$BASE_URL/v1/tasks/<task_id>" \
  -H "Authorization: Bearer $API_KEY" | python -m json.tool

# ── 6. 查询任务列表 ──
curl -s "$BASE_URL/v1/tasks?limit=10&status=succeeded" \
  -H "Authorization: Bearer $API_KEY" | python -m json.tool

# ── 7. 取消任务 ──
curl -s -X POST "$BASE_URL/v1/tasks/<task_id>/cancel" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"reason": "no longer needed"}' | python -m json.tool

# ── 8. 下载产物（获取预签名下载 URL） ──
curl -s "$BASE_URL/v1/tasks/<task_id>/files/final_video" \
  -H "Authorization: Bearer $API_KEY" | python -m json.tool

# ── Admin: 充值配额 ──
curl -s -X POST "$BASE_URL/admin/merchants/<merchant_id>/quota" \
  -H "X-Admin-Key: $ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{"amount": 500, "description": "Initial charge"}' | python -m json.tool

# ── Admin: 查看计费记录 ──
curl -s "$BASE_URL/admin/merchants/<merchant_id>/billing?limit=20" \
  -H "X-Admin-Key: $ADMIN_KEY" | python -m json.tool

# ── Admin: 创建新 API Key ──
curl -s -X POST "$BASE_URL/admin/merchants/<merchant_id>/api-keys" \
  -H "X-Admin-Key: $ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "production"}' | python -m json.tool
架构总结
组件	技术	说明
API 服务	FastAPI + Uvicorn	无状态，可水平扩展
任务队列	Redis + RQ	替代 Thread，支持持久化、重试
对象存储	MinIO (S3 兼容)	产物上传，预签名直传
数据库	PostgreSQL 17	商户/配额/计费/任务持久化
超时看门狗	task_watchdog.py	独立进程定期检查超时任务
商户体系	Merchant + ApiKey 表	多商户隔离
配额计费	预扣→确认/退还	防止超额，失败自动退还
任务取消	Redis flag + RQ cancel	Worker 轮询取消标志
文件直传	presigned PUT URL	客户端直传 MinIO，不经过 API
